二维码生成

二维码生成

- (void)viewDidLoad {
[super viewDidLoad];

// 准备滤镜
CIFilter *filter = [CIFilter filterWithName:@"CIQRCodeGenerator"];

//  设置默认值
[filter setDefaults];

//  生成要显示的字符串数据
NSData *data = [@"这个家伙很懒，什么都没有留下！" dataUsingEncoding:NSUTF8StringEncoding];

[filter setValue:data forKeyPath:@"inputMessage"];

[filter setValue:@"M" forKeyPath:@"inputCorrectionLevel"];

// 输出
CIImage *coreImage = [filter outputImage];

//  1. 要把图像无损放大
UIImage *QRImage = [self imageWithCIImage:coreImage andSize:self.imgView.bounds.size];

UIImage *headImage = [UIImage imageNamed:@"head"];

//  2. 要合成头像
CGSize headSize = CGSizeMake(self.imgView.bounds.size.width * 0.30, self.imgView.bounds.size.height * 0.30);

UIImage *QRCardImage = [self imageWithBackgroundImage:QRImage centerImage:headImage centerImageSize:headSize];

[self.imgView setImage:QRCardImage];

}

// 将CIImage转换成指定大小的UIImage

  • (UIImage )imageWithCIImage:(CIImage )coreImage andSize:(CGSize)size {

    //1. CIImage 转换成 CGImage(CGImageRef)
    CIContext *context = [CIContext contextWithOptions:nil];

    CGImageRef originCGImage = [context createCGImage:coreImage fromRect:coreImage.extent];

    //2. 创建一个图形上下文 Bitmap
    CGColorSpaceRef cs = CGColorSpaceCreateDeviceGray();

    CGContextRef bitmapCtx = CGBitmapContextCreate(NULL, size.width, size.height, 8, 0, cs, kCGImageAlphaNone);

    //3. 将CGImage图片渲染到新的图形上下文中
    CGContextSetInterpolationQuality(bitmapCtx, kCGInterpolationNone);

    // 在图形上下文中把图片画出来
    CGRect newRect = CGRectMake(0, 0, size.width, size.height);

    CGContextDrawImage(bitmapCtx, newRect, originCGImage);

    //4. 取图像
    CGImageRef QRImage = CGBitmapContextCreateImage(bitmapCtx);

    // 释放
    CGColorSpaceRelease(cs);

    CGImageRelease(originCGImage);

    CGContextRelease(bitmapCtx);

return [UIImage imageWithCGImage:QRImage];

}

  • (UIImage )imageWithBackgroundImage:(UIImage )backgroundImage centerImage:(UIImage *)centerImage centerImageSize:(CGSize)centerSize{

    // 开始图形上下文
    UIGraphicsBeginImageContext(backgroundImage.size);

    // 先画背景
    [backgroundImage drawAtPoint:CGPointZero];

    // 再画头像
    CGFloat headW = centerSize.width;
    CGFloat headH = centerSize.height;
    CGFloat headX = (backgroundImage.size.width - headW) 0.5;
    CGFloat headY = (backgroundImage.size.height - headH)     0.5;

    [centerImage drawInRect:CGRectMake(headX, headY, headW, headH)];

    // 取图像
    UIImage *newImage = UIGraphicsGetImageFromCurrentImageContext();

    UIGraphicsEndImageContext();

    return newImage;

}

NSURLSession

NSURLSession

  • NSURLConnection和NSURLSession 区别 ：
    • connection 相对于接触底层，iOS 7.0后苹果推荐使用session
    • NSURLConnection :是面向内存的 ，运行在当前线程消息循环中的
    • NSURLSession :面向内存dataTask 面向磁盘Download Upload
    • con 只有自定义操作才能设置（最大并发数，一来，暂停等）
    • 而session 本身不需要自定义就自带
    • session:自带后台下载和上传功能,可以用另一个进程下载的.

NSURLSession
DataTask(session 会话)
[NSURlSession sharedSession] dataTask(通过shared ,是单例，单独的进程，所有的APP可用,用sharedSession不能配合使用代理，因为一个代理是一对一的)
dataTask默认是挂起的，没有开启需要resume
下载DownloadTask
默认是下载到temp文件夹的，下载完成会自动删除，下载是在子线程上的
//需要赋值到其他位置
[[NSFileManager defaultManager]copyItemAtPath: toPath Error]
跟踪下载进度（从代理方法)
创建session时候，不能用sharedsession，要通过自己创建session,来设置代理,如果设置了代理，就不能用回调的方法，否则代理方法就不会执行。
断点续传
暂停：通过调用cancleByProducingResumeData 方法，方便下次续传时候，使用数据.
bug:当多次点击暂停，再点击继续下载会崩溃。
reason:保存的resumeData 第二次点击后为空，所以在继续时候会错误。
解决方案：sesson置为nil，多次点击就不会讲resumedata 

继续下载：通过downloadTaskwithResumedata：,从保存的位置开始继续下载。  
bug: 1.resumedata为空报错 2.多次点击 都从暂停位置开始下载多次操作 
解决方案：1.设置为空判断 2.点击一次继续下载之后就将session置为Nil
 当停电时候时候，断点续传思路：resumeDate实质是plist，这时候里面存放的文件大小是错误的，但是存放的临时文件是对的，将文件的大小改为临时文件的大小就好了

上传任务
put请求是直接将文件保存到服务器磁盘。（只有webDav能接受PUT方法的请求,只有配置了wabDav的路径，才能接受PUT方法的请求,put请求上传任务相对于post，不用大量拼接请求内容，）

downloadTask（面向磁盘） 下载完成会删除磁盘的内容，如果要保存的话，在Location位置copy到某个文件将爱
dataTask面向内存,默认是有缓存的，cache目录中

session使用的注意事项
session默认是强引用的，在session调用代理时候，会循环引用，使用完后，要让session对象无效，而session调用 invalidate之后， session无法再次使用,置为nil。
configuration 模式 ephemeralSession 一般用于无痕模式

使用NSURLConnectionDataDelagate

  • 通过全局变量，存储文件总大小，实现下载的进度

    • 使用
    • 类似迅雷多线程下载，创建一个同要下载的文件的大小的空文件，内容全部填充为0，文件指针指向

  • 断点续传:
  • 1 .判断是否存在文件
  • 2 .如果文件存不存在就返回0 从开始位置下载

    • 与服务器文件大小相等 返回-1 已经下载完毕
    • 服务器文件的大小，返回文件大小fileSize
    • 本地文件的大小 >服务器文件大小 删除文件，返回0 从开始位置下载

NSURlConnection 默认是在线程的消息循环中执行
的，所以如果要想在子线程中执行，就要在子线程中开启子线程的消息循环，因为子线程的消息循环是默认关闭的。在开启消息循环后，不用管理消息循环的结束，消息循环需要输入源和定时源，下载操作作为输入源，一旦下载完毕，runloop就会缺少输入源而自动关闭
